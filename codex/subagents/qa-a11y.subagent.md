# Subagent — QA e Acessibilidade

## Objetivo

Validar build, comportamento, responsividade e acessibilidade.

## Testes mínimos

- `npm run build`
- `npm run lint`
- abrir `/criar-colaborador` como `MANAGER`
- testar desktop largo
- testar tablet
- testar mobile
- testar CPF vazio
- testar CPF indisponível
- testar CPF disponível
- salvar colaborador
- validar bloqueio/desbloqueio do acesso
- testar username indisponível
- testar username disponível
- concluir cadastro
- verificar reset do fluxo

## Acessibilidade

- labels explícitos;
- foco visível;
- botões com texto;
- status com texto;
- erro perto do campo;
- CTA fixo no mobile com área de toque suficiente.
